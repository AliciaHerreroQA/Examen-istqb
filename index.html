Examen ISTQB

1. ¿A qué actividad da soporte una herramienta de preparación de datos de prueba?

A) Monitorización y control de prueba. B) Análisis y diseño de la prueba. C) Implementación y ejecución de la prueba. Sin Responder

2. Which item correctly identifies a potential risk with test automation?

A) Test tools may not be reliable enough. B) Unrealistic expectations regarding functionality of a tool. C) There may be unknown side effects in operational use. Sin Responder

3. Which of the following is NOT an example of the shift left approach?

A) Create component test cases before programming the associated code. B) Execution of non-functional tests, starting during system level testing. C) Executing a performance efficiency test for a component during component testing. Sin Responder

4. Your organization’s test strategy suggests that once a system is planned for retirement, data migration to a new system shall be tested. As part of what test activity is this testing MOST likely to be performed?

A) Maintenance testing. B) Regression testing. C) Component testing. Sin Responder

5. Which of the following is a benefit of early and frequent feedback by stakeholders?

A) It improves the test process for future projects. B) It is the only way to measure the quality of changes. C) It helps avoid requirements misunderstandings. Sin Responder

6. Which of these statements is NOT a factor that contributes to successful reviews?

A) Management´s personal involvement in communicating findings. B) Establishing clear objectives and measurable exit criteria for the review. C) Participants should dedicate adequate time for the review. Sin Responder

7. Which of the following statements is NOT true for white-box testing?

A) White-box testing is testing based on the analysis of the internal structure of a component or system. B) White-box coverage metrics can help identify additional tests to increase code coverage. C) White-box testing can help identify unimplemented requirements. Sin Responder

8. Which of the following activities describes the BEST way to formulate acceptance criteria for a given user story?

A) Performing retrospectives to determine the actual needs of the stakeholders. B) Using the given/when/then format to describe an example test condition C) Documenting risks related to a given user story in a test plan to facilitate the risk-based testing Sin Responder

9. How do testers add value to iteration and release planning?

A) Testers determine the priority of the user stories to be developed. B) Testers participate in risk analysis and determine testability of user stories. C) Testers focus only on refining the functional aspects of the system under test. Sin Responder

10. Which ONE of the following options is an exit criterion for testing a system?

A) Expected defect density is reached and defects are reported. B) Test environment readiness. C) Requirements are translated into given/when/then format. Sin Responder
